---
title: "Measurable improvement for AI-operated systems"
type: "vision"
date: "2026-04-01"
---

An AI agent operating a system cannot improve what it cannot measure. loss-forge teaches agents to define loss functions for any project — not just ML training loss, but operational loss: is the system accomplishing its missions? Is the codebase getting simpler or more complex? Is the agent getting faster or slower? Are the results getting better or worse?

Every project has end missions (what it exists to do) and system health metrics (is anything broken). loss-forge provides the skills to define both, measure baselines, and track improvement over time. The output is an instrument panel — one command that tells you the composite loss and mission completion score.

The forge doesn't prescribe what to measure. It teaches agents HOW to measure, and enforces that every change is evaluated against the numbers. If loss goes up without justification, the change is wrong.

Success: any agent can clone a project, run the loss command, see the numbers, make a change, re-run, and know immediately whether they helped or hurt.
